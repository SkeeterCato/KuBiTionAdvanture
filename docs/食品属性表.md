# 食品烹饪方法与交易价值

数据来源：

- 配方：`src/data.js` 的 `COOK_DATA`
- 食品效果：`src/data_item.js` 的 `ITEM_DATA[*].effect`
- 烹饪匹配：`src/main.js` 的 `StudioComponent.getCookResult`
- 交易价值：`src/main.js` 的 `getValue`

规则备注：

- `COOK_DATA` 原始配方 74 条，去掉完全重复的材料组合后是 72 种做法。
- 烹饪匹配从 `COOK_DATA` 末尾向前查找；同一材料组合如果有重复规则，靠后的规则生效。
- 交易价值优先使用 `ITEM_DATA[id].value`；没有直接价值时，食品按正向状态收益计算，其中 `ps` 按 0.5 倍，`temp` 和负数效果不计入。
- `flour` 出现在两条烹饪配方里，但当前 `ITEM_DATA` 没有 `flour`，所以材料交易价值无法计算。
- 单材料配方虽然写在 `COOK_DATA` 里，但当前烹饪锅少于 2 种物品会失败，实际不能只放一种材料触发。
