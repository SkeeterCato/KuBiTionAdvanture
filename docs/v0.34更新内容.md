## BUG / 数值修复 SPEC

### Problem Statement

现有数据和逻辑有几处不一致：

- 废弃的 `flour` 烹饪配方仍然保留，但没有对应物品和来源。
- 酒精、降温药剂的交易价值和当前设计不一致。
- 降温药剂只有降温效果，没有体现冰露带来的水分补充。
- 狙击枪和精锐狙击枪描述区分度不足。
- 死灵法杖描述有尸体数量上限，但代码没有执行。
- 缺少与降温药剂对称的升温药剂。

### Proposed Solution

- 删除 `COOK_DATA` 中两条 `flour` 配方，不新增 `flour` 物品。
- 酒精固定价格改为 20。
- 降温药剂固定价格改为 20，效果改为 `moist:+10,temp:-35`，描述改为“补充水分并快速降低体温”。
- 新增 `firePotion` 升温药剂：
  - 固定价格 20。
  - 效果为 `full:+10,temp:+35`。
  - 描述为“可以充饥并快速提高体温”。
  - 在炼金台制作，炼金提纯(I)解锁，耗时 1。
  - 配方为 `fire x2`。
- 狙击枪描述改为“高射程的远程枪械武器，需要子弹。”
- 精锐狙击枪描述改为“超高射程的精锐枪械武器，需要子弹。”
- 死灵法杖按最多 1000 个 `humanMeat` 计算加伤，最多增加 2000 伤害。

### Technical Constraints

- 本批改动作为 `v0.34` 的 BUG / 数值修复内容。

- 文本变更需同时更新数据表文档。

  - `node --check dist/itch-html5-v0.34-flat/main.js`
  - `node tests/mobile-itch-regression.test.js`
