# 已知问题

## 同名武器共用耐久

状态：待解决。

现象：玩家制造或获得一把同名武器时，新武器可能显示为非满耐久。

原因：当前 `durableSaveData` 按物品 ID 记录耐久损耗，不按单件武器记录。制造逻辑只增加物品数量，不会重置该 ID 的耐久损耗。

影响：同名武器堆叠后共享耐久状态，玩家会感觉新制造的武器继承了旧武器的损耗。

相关代码：

- `src/data.js`：`DURABLE_INIT`
- `src/main.js`：`StudioComponent.make`
- `src/main.js`：`changeItem`
- `src/main.js`：`durableChange`

处理方向：需要先决定是否允许同名武器继续堆叠。如果继续堆叠，只能维持按 ID 共享耐久；如果要每把武器独立耐久，需要调整背包数据结构和存档兼容。

## 死灵法杖尸体加伤没有上限

状态：待解决。

现象：死灵法杖描述为“每控制一个尸体（上限1000），伤害便加2”，但实际伤害按背包尸体数量无限增加。

原因：`src/main.js` 的 `BattleComponent.getPlayerDamage` 中，`deadStaff` 分支直接把 `humanMeat * 2` 加到伤害上，没有限制上限。相邻的 `knifeStaff` 分支有 `amount > 1000 ? 1000 : amount` 限制，说明这里很可能漏了同类限制。

影响：玩家囤积大量尸体后，死灵法杖伤害会超过描述上限，破坏武器平衡。

相关代码：

- `src/data_item.js`：`deadStaff.desc`
- `src/main.js`：`BattleComponent.getPlayerDamage`

处理方向：如果遵守物品描述，应把 `deadStaff` 的尸体计数限制在 1000 个尸体以内，即最多增加 2000 点基础伤害；如果要保留无限成长，需要同步修改物品描述和武器文档。

## 远程武器耐久被提炼科技提高

状态：待解决。

现象：`提炼(I/II/III)` 的描述是增加魔法武器耐久，但当前代码会让远程武器也受益。

原因：`src/main.js` 的 `getMaxDurable` 只区分 `weaponType == 'melee'` 和其他类型。近战武器使用 `durableUpdate`，所有非近战武器都会使用 `magicDurableUpdate`，因此 `shoot` 远程武器也被归入提炼科技。

影响：科研分类和描述不一致。远程武器没有自己的耐久成长路线，实际却占用了魔法武器的提炼路线。

相关代码：

- `src/main.js`：`getMaxDurable`
- `src/data_item.js`：`durableUpdate_1/2/3`、`magicDurableUpdate_1/2/3`
- `src/data_studio.js`：`durableUpdate_1/2/3`、`magicDurableUpdate_1/2/3`

处理方向：新增远程武器耐久科研树，命名为 `工艺(I/II/III)`，只作用于 `weaponType: 'shoot'`。之后 `getMaxDurable` 应按 `melee`、`shoot`、`magic` 分别读取锻造、工艺、提炼等级。
