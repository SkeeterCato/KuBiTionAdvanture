# v0.35 更新内容

## 界面体验 SPEC

### Problem Statement

当前 PC 和手机在陷阱、背包、交易、采集、制造、整理上有几处体验问题：

- PC 悬浮陷阱不会在右侧详情区显示陷阱信息。
- 手机陷阱缺少明确拆除入口。
- 手机背包、大箱子、商人交易、任务提交等场景无法同时看目标区和背包，点物品又容易直接转移或提交，无法先看详情。
- 大箱子手机布局需要横向滑动。
- 采集后未拾取资源会因误触返回丢失。
- 制造详情页看不到当前已有成品数量。
- 背包缺少手动整理，装备查找不方便。

### Proposed Solution

- PC 陷阱：
  - 悬浮已设置陷阱时，右侧详情区显示陷阱名、状态、诱饵/需求、收益。
  - 未触发陷阱额外显示“右键拆除”标签。
  - 已触发陷阱显示详情，但不显示拆除标签。
  - 鼠标离开陷阱后详情保留，直到悬浮其它物品或状态栏覆盖。
- 手机陷阱：
  - 未触发陷阱行内显示“拆除”按钮。
- 手机物品交互：
  - 所有涉及背包物品选择的场景，采用上方目标区、下方背包区的同屏布局。
  - 目标区和背包区各自纵向滚动。
  - 物品网格自适应宽度，不横向滚动。
  - 点物品只选中并显示详情，详情里显示当前场景可用操作按钮，例如放入、取出、使用、出售、购买、提交。
  - PC 保持现有快捷操作。
- 采集资源：
  - 保持现有采集结果区和“全部拾取”UI，不增加新 UI。
  - 采集出的待拾取资源保存到当前地点状态里。
  - 返回后再进入同一地点，资源仍显示。
  - 点“全部拾取”后，成功放进背包的部分清空；背包装不下的部分继续留在当前资源区。
- 制造详情：
  - 在制作确认框里新增“拥有”列。
  - 只显示目标成品数量，统计背包 + 大箱子。
- 整理：
  - 背包标题栏旁增加“整理”按钮。
  - 背包和大箱子整理时，排序改为已装备物品最前，然后其它装备类，然后其它物品。
  - 装备类包括 `type:'weapon'` 或有 `equipType` 的物品。
  - 非背包/大箱子容器不变。

### Technical Constraints

- 作为 `v0.35` 的界面体验内容。
- 从 `dist/itch-html5-v0.34-flat/` 复制生成 `dist/itch-html5-v0.35-flat/`，旧版 flat 产物保留。
- 改源码后同步 `dist/itch-html5-v0.35-flat/`。
- 发布前重建 `dist/超苦逼冒险者-v0.35-html5-flat.zip`，ZIP 根目录保持 flat bundle 结构。
- 不新增新的采集暂存 UI。
- PC 原有左键/右键快捷操作尽量保持。
- 手机按钮必须保持 `type="button"` 并抑制默认点击行为。
- 验证覆盖 PC 和手机强制布局，至少包括现有 mobile regression。

### Non-goals

- 不重做背包数据结构。
- 不处理每把武器单独耐久。
- 不新增地点仓库或资源暂存仓库。
- 不改变采集资源产出公式。
- 不改变商人交易价格规则。

### Success Criteria

- PC 悬浮陷阱能在右侧详情区看到陷阱信息；未触发陷阱能看到“右键拆除”。
- 手机可以通过明确按钮拆除未触发陷阱。
- 手机在背包、大箱子、交易、任务提交等物品选择场景能同时看到目标区和背包区，能先看详情再操作。
- 手机大箱子物品网格不需要横向滑动。
- 采集后未拾取资源不会因返回再进入而丢失。
- 制造确认框能看到目标成品当前拥有数量。
- 背包可以手动整理，背包和大箱子整理后装备类排在前面，已装备物品最前。
- 现有 mobile regression 通过。
